Advanced Git Commands Every Developer Must Know

Tue 23 March 2021

Git, as you may already know by now, is a Distributed Version Control System. It was created by Linus Torvalds in 2005 and is currently the most widely used source-code management tool by professional software developers.

Although there are multiple GUIs available that allow developers to work with Git, using the Git commands in the command line is till the most powerful and flexible way to use Git. Here are some advanced commands to help you manage your source code better.

1. Switching between most recent branches

git checkout -

One of the more intuitive and niftier shortcuts in git is the git checkout -. The checkout git command, as you may already know, is used to switch between branches (among other things), the - after it takes you to the most recent previously used branch. It's really nifty because you don't really need to type in the full branch name and struggle with options in the tab completion and is a great way to switch quickly. It's intuitive because, and veteran unix command line users will have already noticed this, it seems to be inspired from - argument to the cd command.

1. Pick those cherries

git cherry-pick <COMMIT ID>

If you do not know what cherry-pick is, it's a git command that allows you to apply the changes introduced by one or more commits in one branch onto another branch.

You can also cherry pick multiple commit (which could be on another branch) using the command;

git cherry-pick <COMMIT ID A>..<COMMIT ID B>

1. List all Git branches chronologically

git branch --sort=-committerdate

Have you been in situations where you have revisited a project after a long while and just cannot remember the most recent branches you were working on. This command is exactly what you need.

The command lists branches in descending order with the branches that have most recent commits coming first. You can use --sort=committerdate to get results in ascending order.

Note that this works for Git version 2.7.0 and above. For people unfortunate enough to be stuck with older versions you can use git for-each-ref --sort=-committerdate refs/heads/ which also specifies the commit ID (this command works in Git 2.7.0 too).

1. Finding Deleted Files

There have been cases where I want to find one or more files that I've deleted from a project in the past. Depending on what you need, you can user one of the following command;

When you remember the exact path to the deleted file;

git log --all -- <PATH-TO-FILE>

When you are not so sure of the file path;

git log --all -- **/<FILENAME>

In the command above you can also use wildcard patterns in the filename in case you are not sure of the complete filename.

And when you are not so sure of the path or the filename, try to sift through all the deleted files as give by this command;

git log --diff-filter=D --summary
